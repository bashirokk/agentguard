# Developer community launch checklist

## Before posting

- Publish `v0.1.0` with checksums and release notes.
- Confirm PyPI and Docker installation commands in clean environments.
- Confirm status badges and enable private vulnerability reporting.
- Seed good first issues and a welcome Discussion.
- Prepare one short terminal recording or screenshot without real credentials.
- Monitor security reports privately and general feedback publicly.

## Tailored messages

- **OWASP/LLM security groups:** lead with threat model, rule precision, and request review of categories.
- **LangChain/CrewAI/AutoGen communities:** show one framework-specific vulnerable/safe example and ask for patterns.
- **MCP community:** focus on wildcard tool grants, filesystem roots, and input schemas.
- **DevSecOps communities:** lead with SARIF, exit thresholds, deterministic offline use, and plugin policy packs.
- **Python communities:** lead with package design, typed rule API, contributor tasks, and supported Python versions.

Avoid cross-posting identical text at the same time. Answer technical questions directly, document limitations, and convert repeated feedback into public issues.
